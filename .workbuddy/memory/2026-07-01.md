## 布局修复：三列底部对齐（posts/[...page].astro）

### 问题
- 分页组件（pagination nav）嵌在中间 `<section>` 内部，用 `mt-auto` 推到底部
- 导致中间列高度包含分页组件，三列 stretch 对齐时底部是对齐了分页组件底部而非内容区底部

### 方案：CSS Grid 双行布局
- Grid 容器添加 `lg:grid-rows-[minmax(0,1fr)_auto]`，显式定义两行
  - Row 1 (`minmax(0,1fr)`)：三列内容区，`items-stretch` 让容器底部对齐
  - Row 2 (`auto`)：分页组件，仅占中间列
- 将分页 `<nav>` 从 `<section>` 内部移出，作为 Grid 直接子元素
- 分页 nav 添加 `lg:col-start-2 lg:row-start-2` 精确定位到第2行第2列
- 移除 `mt-auto pt-3 pb-3`（Grid gap 已提供间距）

### 第二次修复：三列内容底部对齐
- 问题：左右侧栏的 `flex-1` 空 div 放在列末尾，将前面所有内容推到顶部，导致左右列最后一个内容块（热门文章 / RSS 订阅）和中间列最后一张卡片不在同一水平线
- 方案：将 `flex-1` 从列末尾移至**最后一个内容块之前**
  - 左列：`flex-1` 放在"最新文章"和"热门文章"之间 → 热门文章被推到底部
  - 右列：`flex-1` 放在"技术关键词"和"订阅"之间 → 订阅被推到底部
  - 中间列：最后一张卡片自然到底部
- 结果：三列最后一个内容块（热门文章、最后一张卡片、RSS 订阅）在水平方向底部对齐，分页组件在 Grid 第 2 行独立排列
