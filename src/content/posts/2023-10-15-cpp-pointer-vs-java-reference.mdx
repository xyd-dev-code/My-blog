## 起因

上学期啃完 C++ Primer 前 8 章，对指针的理解刚到"能跑通链表作业"的水平。这学期开 Java，第一节课老师就甩了一句：

> "Java 没有指针，所有对象都是引用。"

我当场懵了——既然没有指针，那 `new` 出来的东西到底存在哪？为什么传对象进方法，方法里改完外头也跟着变？这一篇就是把我当时的困惑和最终想明白的过程记下来。

## 先看现象：C++ 和 Java 看起来很像

```cpp
// C++
int* p = new int(10);
cout << *p;  // 10
```

```java
// Java
MyClass obj = new MyClass();
obj.doSomething();
```

**乍一看都是「指向」某个东西，但本质完全不同。**

## 三个本质差异

### 1. C++ 的指针是个整数，Java 的引用是个句柄

C++ 里指针的本质就是 **内存地址**，可以打印、可以做加减运算：

```cpp
int a = 10;
int* p = &a;
cout << p;   // 0x7ffee4bff5ac，真的能打印地址
cout << p+1; // 地址 + 4（int 占 4 字节）
```

Java 里 `obj` 是个 **引用**，你可以理解成 JVM 内部维护的一个"句柄 ID"，但你拿不到这个数字，也不能算术：

```java
MyClass obj = new MyClass();
// System.out.println(obj + 1);  // 编译报错，引用不是数字
```

> 类比：C++ 指针像门牌号，Java 引用像房卡。门牌号你能看见数字，房卡只能用来开门。

### 2. 内存管理：手动 vs 自动

C++ 需要 `new` 配 `delete`，漏掉就内存泄漏，写错时机就悬空指针：

```cpp
int* p = new int(10);
// ... 中间某处忘记 delete，内存就漏了
// 或者 delete 后继续用 p = 悬空指针，行为未定义
```

Java 靠 **GC（Garbage Collector）** 自动回收：

```java
Object obj = new Object();
obj = null;  // 原来那个对象没引用了，等 GC 来收
```

刚开始写 Java 的时候，我下意识还在想"这个对象什么时候 delete"，后来强迫自己戒掉这个念头——**你只管 `new`，JVM 兜底**。

### 3. 能不能指向「空」

C++ 里有著名的「**野指针三件套**」：

- 空指针 `nullptr`
- 悬空指针 `delete` 后没置空
- 未初始化指针（最危险，不知道指向哪）

Java 里没有野指针这回事——**没初始化的引用是 `null`，访问会抛 `NullPointerException`，而且是运行时立刻抛**，不会像 C++ 那样静悄悄地乱跑。

```java
String s;
System.out.println(s.length());  // 编译都过不了，提示可能未初始化
```

## 一段代码看懂「Java 传引用」

```java
public static void main(String[] args) {
    int[] arr = {1, 2, 3};
    modify(arr);
    System.out.println(arr[0]);  // 999，不是 1
}

static void modify(int[] a) {
    a[0] = 999;
}
```

我当时在这里卡了很久：C++ 里数组传参会退化成指针，改 `a[0]` 外头确实会变，这我能理解。但 Java 没指针，为什么也变了？

答案是：**`arr` 是个引用，指向堆里的数组对象。传进方法的是这个引用的拷贝，但拷贝和原件指向同一个对象。**

图示：

```
main 栈:        arr ──────┐
                          ▼
堆:          [ 1, 2, 3 ]   ← 同一个数组对象
                          ▲
modify 栈:   a ────────────┘
```

> 关键点：**Java 永远按值传递。** 对象传的是引用的值（拷贝），基本类型传的是数字本身。

## 当时踩的坑

**坑 1：以为 `=` 是复制对象**

```java
MyClass a = new MyClass();
MyClass b = a;  // b 和 a 指向同一个对象，不是复制
b.name = "改了";
System.out.println(a.name);  // "改了"
```

**坑 2：在循环里 `new` 完不释放**

写 C++ 时养成的强迫症，总觉得"对象一直 `new` 是不是在泄漏"。其实 Java GC 会处理，唯一要注意的是 **别让对象被意外引用**（比如静态集合装业务对象），那种情况 GC 也救不了你。

**坑 3：试图把 `null` 当成 "0 号地址"**

C++ 里 `NULL` 是 `0`，但 Java 的 `null` 只是"这个引用没指向任何对象"，**不是地址 0，也不是 0**。`null instanceof Object` 是 `false`。

## 给和我一样新手的建议

1. **不要再画内存地址图了**。Java 里"引用指向对象"够用，不要去脑补 `0x00007f...`。
2. **每次遇到 "为什么外头也变了" 的时候，先判断是基本类型还是引用类型**。基本类型不会变，引用类型会。
3. **`==` 比较引用，`equals` 比较内容**。这个下一篇讲集合的时候还会再撞上。
