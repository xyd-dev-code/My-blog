之前写过电脑商城项目、写过面试题，但**真到生产环境写过代码之后**，**对"学习"这件事的想法完全不一样了**。

## 真实感受：和学校的差距

实习前以为"**学校项目已经够好了**"。

实习后真正感受到的差距：

### 1. 细节不是"吹毛求疵"

| 我之前认为 | 实际意义 |
|------------|----------|
| "注释不重要，能跑就行" | **线上排查靠注释** —— 没注释的代码大家不敢动 |
| "异常直接 print" | **异常分类是为了用户体验 + 监控定位** |
| "日志打不打无所谓" | **requestId 是救命的 —— 故障时全靠它** |
| "try-catch 没必要，麻烦" | **全局异常是为了让代码更清晰** |
| "这个参数简化下" | **统一接口是为了降低协作成本** |

**学到的** —— **每个"啰嗦"的写法背后都有理由**。

### 2. 写代码不是"程序员"的全部

之前以为程序员就是"坐下来敲代码"。

**实际工作**：

- **沟通**：和导师、产品、前端、测试、运维都聊
- **看代码**：50% 时间看别人代码
- **文档**：写 wiki、改 README、补接口文档
- **排错**：30% 时间在 debug 和查日志
- **开会**：周会、立会、需求评审、代码评审
- **自我学习**：新技术、规范、工具

**真正写代码时间可能只有 30%**。

### 3. 工具链比想象中重要

公司里：

- 飞书（沟通、看日历、读 wiki）
- GitLab（MR、Code Review、CI）
- Jenkins（发版）
- Grafana / Prometheus（监控）
- ELK（日志）
- DBA 平台（慢 SQL）
- Jira-like 工单系统
- Apifox（接口文档）
- VPN / 堡垒机

**每个都有学习成本**。新人入职前两周基本都在学这些。

### 4. "工程化"不是嘴上说说

公司里每件事都"流程化"：

- 改代码 → 开分支 → MR → Code Review → CI 通过 → 合并 → 上预发 → 监控 → 上生产
- 修 bug → 工单 + 修复 + 单测 + 复盘文档
- 加索引 → DBA 工单 + 审核 + 低峰期执行 + 回滚方案
- 发功能 → 需求评审 + 技术方案 + 工时评估 + 测试 + 上线

**任何一个步骤都不能跳** —— **跳了就出问题**。

### 5. "会 Java"和"能做工程"是两回事

W 哥让我看一段话，**我觉得是这 4 个月学到的最核心的**：

> "**会 Java** —— 是知道每个 API、每个框架怎么用。
> **能做工程** —— 是知道什么场景用什么方案，怎么和团队协作，怎么保证代码质量，怎么让系统稳定。
> 两个是完全不同的能力。"

我之前是前者。

**实习之后正在努力做到后者**。

## 给后辈实习生的真诚建议

### 1. 别上来就追"新技术"

之前我简历里写了一大堆技术栈 —— Spring Cloud、Kafka、Elasticsearch、Docker、K8s……

**面试官的问题** —— "**这个你用过吗**？" —— "**看过文档，跑过 demo**。"

W 哥说：

> "**'看过文档'和'用过'是两个概念**。**找一个项目** —— **把一项技术用熟** —— **简历上写'用熟'比写'知道'值钱**。"

**我现在的做法**：

- 简历只写"**深入用过**"的：SpringBoot + MyBatis + MySQL + Redis
- 加上一句话"**电商场景 SpringBoot 完整项目**" + "**生产环境 SQL 优化 16 倍**"
- 面试时能讲出"**具体怎么做的、踩过什么坑**"

**比写一堆技术名词有用 100 倍**。

### 2. 实习经历怎么写简历

**写之前**：  
"实习期间参与了 XX 系统的开发，主要负责用户模块的实现。"

**写之后**（改了三遍）：

> **亚信科技有限公司 | 后端开发实习生 | 2025.12 - 2026.03**
>
> - 参与业务支撑系统后端模块维护，基于 Spring Boot + MyBatis 实现用户鉴权与权限控制
> - 处理 P2 级生产工单：定位并修复用户列表 NPE，加 Optional 防御 + 单元测试覆盖
> - 主导慢 SQL 优化：分析 800ms 慢查询，添加联合索引 `(org_id, status, create_time)`，配合预发 sysbench 压测，P95 从 800ms 降至 50ms
> - 编写与维护接口文档（Apifox），配合前端完成联调测试；参与代码评审，沉淀 3 篇索引规范文档

**对比**：

| 写之前 | 写之后 |
|--------|--------|
| "参与了 XX 系统开发" | "处理 P2 级工单" |
| "做了 XX 模块" | "定位并修复 NPE" |
| "优化了 SQL 性能" | "**P95 从 800ms 降至 50ms**" |
| "写了文档" | "沉淀 3 篇索引规范文档" |

**每个项目都要有"数字" + "具体动作" + "业务价值"**。

### 3. 面试怎么讲实习经历

**常见问题**："**介绍下你实习做的最有挑战的事**"

**我面试时的回答版本**：

> "**我处理过一个慢 SQL 优化的工单** —— **用户列表接口 P95 800ms**。
>
> 排查过程分三步：
> 1. **DBA 平台慢 SQL 日志定位**到一条 `WHERE org_id = ? AND status = ? ORDER BY create_time` 的查询
> 2. **EXPLAIN 看到 type=ALL，扫了 1.2 万行**，因为 `org_id` 没有索引
> 3. **加联合索引** `(org_id, status, create_time)`，**预发环境 sysbench 压测**：P95 从 1200ms 降到 22ms
>
> 生产验证：**P95 从 800ms 降到 50ms**。
>
> 学到的：**SQL 优化要看 EXPLAIN、执行计划、压测对比，不能光看监控**。"

**面试官追问**：

- "**为啥用联合索引而不是单列索引？**" —— **基数 + 覆盖 WHERE + ORDER BY**
- "**为啥不直接改 SQL 重写？**" —— **老系统动 SQL 风险大，加索引动数据少**
- "**怎么验证？**" —— **sysbench 100 并发 60 秒，对比前后 P95/P99**
- "**回滚方案？**" —— **DROP INDEX，5 秒回滚**

**每个追问都有答案** —— **这才是"真做过"**。

### 4. 大四上就开始找实习

我 2025 年 9 月开始投简历，投的是"2026 届实习"。

**建议**：

- **大四上 9-10 月**投实习岗（很多公司全年招实习）
- **大四下 2-3 月**结束实习，**4-6 月毕业**
- **毕业前 1 年**关注秋招（**注意**"25 届秋招"通常指 2025 年毕业的同学，"25 届春招"是补录）

**不要拖到大四下才找实习** —— **那时候已经晚了**。

### 5. 第一份实习"公司名"比"岗位"重要

**我自己的观点** —— 仅代表个人感受：

- **大厂实习**（阿里、字节、腾讯、美团...）：**好找工作**，**有体系培训**
- **中型公司实习**（亚信、一些垂直行业龙头）：**真能干活**，**接触全面**
- **小公司实习**：**可能就 1-2 个后端**，什么都是你干

**亚信是中型公司** —— **不是互联网一线**，但：

- 客户都是大企业（运营商、银行）
- 项目流程规范
- 不会让实习生打杂
- 学到的东西可以写进简历

**如果你只能去小公司** —— **也别挑**，**有实习经历比没有强**。

### 6. 实习中多主动

我观察周围同事（正式员工），主动和不主动差距很大：

| 类型 | 表现 |
|------|------|
| 主动型 | 导师讲一句"我们有用这个框架" → 主动去看文档；导师说"有空帮我看看这个" → 主动承担；有问题主动问；做完主动汇报 |
| 被动型 | 导师布置了才做；不问就不做；做完不汇报 |

**两种人半年后完全不一样**。

W 哥说我"主动型"已经合格，**叫我"放心"**。

### 7. 写日报很重要

**不要觉得是形式主义**。

日报帮我：

- **理清今天做了啥**（周五写周报就有素材）
- **记录踩过的坑**（后面写博客用得上）
- **让导师知道我在干活**（不写导师以为我没做事）
- **实习结束时翻翻**（写简历用）

**我 4 个月写了 80 篇日报**（每周 5 天，4 个月）—— **现在翻回去**，**能回忆起每个细节**。

### 8. 写博客要写真东西

之前博客写的"xxx 面试题" —— **看了就忘**。

真正有用的博客 —— **是写自己怎么踩坑的**：

- **某个工单怎么处理的**
- **某个工具怎么装的**
- **某个 bug 怎么修的**
- **某段代码为什么这样写**

**这篇博客就是例子** —— **全是真实经历**。

### 9. 简历上不要撒谎

面试官会追问，**一追问就露馅**。

我简历写"了解 SpringBoot" —— 我真的了解，**能讲清楚启动流程、IoC、AOP**。

**绝不写**：

- "**精通**"（面试官一问三不知就死）
- "**熟悉 K8s**"（只跑过 docker run）

**老老实实写"了解/熟悉/掌握"**。

### 10. 实习时记得"带脑子上班"

W 哥说我最大优点是"**带脑子**" —— **不是夸**，**是观察**：

- 看代码会问"**为啥这样写**"
- 改 bug 会问"**为啥会触发**"
- 学 SQL 会问"**为啥这个索引这样建**"

**这就是成长** —— **不是"完成任务"**，**是"想明白为啥这样做"**。

## 实习中我犯过的几个错误

### 1. 第一次发版前没跑通

第一次走完整 Git Flow 时，**我直接 push 了** —— **没本地 build 验证**。

结果 CI 红了，**W 哥脸色难看**：

> "**本地不跑，CI 红了浪费大家时间**。"

**改**：push 前必须本地 `mvn clean install` 通过。

### 2. Code Review 没认真看

第一次提交 MR，**reviewer W 哥提了 5 个意见** —— 我改了 3 个，**还有 2 个觉得"无所谓"**。

后来才知道**那 2 个"无所谓"的问题后续真的出过 bug**。

**改**：Reviewer 的意见**逐条认真回复**，**哪怕不采纳也要说明理由**。

### 3. 没及时同步进度

有一次做了 3 天没给 W 哥汇报，**他以为我没进展**。

**改**：每天日报 + 关键节点即时汇报。

### 4. 自作主张改 SQL

有一次为了"简化"，**擅自把 SQL 重写了** —— **没经过 W 哥**。

结果 SQL 没优化到，反而多扫了 30% 表。

**改**：SQL 改动前必须 reviewer 同意。

## 真正的"工程化思维"

实习学到最有价值的是"**工程化思维**"。

不只是"会写代码" —— 是"**考虑全局**"。

**例 1**：用户列表查询

**学生思维**：写个 SQL 就行。

**工程思维**：

- 这条 SQL 上生产会出多少数据？要不要分页？
- 如果数据库挂了怎么办？要不要加缓存？
- 这个查询在哪调用？批量调用怎么优化？
- 这个接口响应慢，用户体验怎样？
- 上线后怎么监控？

**例 2**：加一个字段

**学生思维**：entity 加字段 + 加数据库 migration。

**工程思维**：

- 字段对老数据有影响吗？
- 字段对接口有影响吗？（要不要兼容老版本）
- 字段加索引吗？
- 字段需要前端配合吗？
- 改完走完整上线流程吗？

**这种"全局观"是实习学到的最大财富**。

## 给学校教学的一点建议

不是我有资格评价教学，**只是个人感受**：

1. **真实项目比"轮子项目"重要** —— 学校教 SpringBoot 是对的，但能做成完整电商比写一堆 demo 有用
2. **生产实践课** —— 哪怕去小企业实习一学期，也比课堂模拟强
3. **SQL 优化** —— 这个学校基本没教，但生产天天用
4. **Git Flow + 代码评审** —— 实际工作每天用，学校可以加门课
5. **日志 + 监控 + 排查** —— 这些完全没教过

**但也别怪学校** —— **学校教基础，**工程化靠实习补**。
