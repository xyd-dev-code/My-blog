## 起因

之前学的：

- 后端 SpringBoot + MyBatis-Plus（返回 JSON 接口）
- 前端 Vue 3 组件（写页面）

但这两套是**独立的**，没串起来。

老师布置作业：

> "前端调后端接口，实现用户列表的展示、登录、增删改查。"

我以为装个 Axios 就完事了，**没想到踩了 6 个坑**。这篇记一下。

## 准备工作

我有两个项目：

- **后端**：`mall-backend`（SpringBoot，端口 8080，提供 `/api/users` 接口）
- **前端**：`mall-frontend`（Vue 3 + Vite，端口 5173）

前后端分离，**不同端口**，问题就来了。

## 第一步：装 Axios

前端项目目录：

```bash
npm install axios
```

写一个最简单的调用：

```javascript
// src/api/user.js
import axios from 'axios';

export function listUsers() {
    return axios.get('http://localhost:8080/api/users');
}
```

Vue 组件调用：

```vue
<template>
  <ul>
    <li v-for="user in users" :key="user.id">
      {{ user.username }}
    </li>
  </ul>
</template>

<script setup>
import { ref, onMounted } from 'vue';
import { listUsers } from '../api/user';

const users = ref([]);

onMounted(async () => {
    const res = await listUsers();
    users.value = res.data.data;  // 后端返回 {code, data, message}
});
</script>
```

浏览器控制台：

```
Access to XMLHttpRequest at 'http://localhost:8080/api/users' 
from origin 'http://localhost:5173' has been blocked by CORS policy: 
No 'Access-Control-Allow-Origin' header is present on the requested resource.
```

**第一个坑：CORS 跨域**。

## 第一个坑：CORS 跨域

**为什么跨域？**

- 前端 `http://localhost:5173`
- 后端 `http://localhost:8080`
- **协议相同、域名相同，但端口不同**——浏览器认为"跨域"

浏览器默认不允许跨域请求，要服务端允许。

### 解决方案 A：后端加 CORS 配置（推荐）

后端 `WebConfig.java`：

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {

    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/**")  // 所有路径
                .allowedOrigins("http://localhost:5173")  // 允许这个源
                .allowedMethods("GET", "POST", "PUT", "DELETE")
                .allowedHeaders("*")
                .allowCredentials(true)  // 允许带 cookie
                .maxAge(3600);
    }
}
```

或者用注解（更细粒度）：

```java
@RestController
@RequestMapping("/api/users")
@CrossOrigin(origins = "http://localhost:5173")
public class UserController {
    // ...
}
```

### 解决方案 B：Vite 代理（前端解决）

我更喜欢这种——**前端解决跨域问题，后端不用改**。

`vite.config.js`：

```javascript
import { defineConfig } from 'vite';
import vue from '@vitejs/plugin-vue';

export default defineConfig({
    plugins: [vue()],
    server: {
        proxy: {
            '/api': {
                target: 'http://localhost:8080',
                changeOrigin: true
            }
        }
    }
});
```

**原理**：

- 前端发请求到 `/api/users`（同源）
- Vite 代理把请求转发到 `http://localhost:8080/api/users`
- 后端响应，Vite 转发回前端
- **浏览器看是同源请求，不触发 CORS 检查**

前端代码改成：

```javascript
// 不写完整 URL，写 /api
export function listUsers() {
    return axios.get('/api/users');
}
```

**两种方式都能用**：

- **方案 A** 适合前后端独立部署
- **方案 B** 适合开发环境（不用动后端）

我选 B，省事。

## 第二步：登录功能

后端登录接口：

```java
@RestController
@RequestMapping("/api/auth")
public class AuthController {

    @Autowired
    private UserService userService;

    @PostMapping("/login")
    public Result<String> login(@RequestBody LoginRequest req) {
        String token = userService.login(req.getUsername(), req.getPassword());
        return Result.ok(token);
    }
}
```

前端登录页面：

```vue
<template>
  <div>
    <input v-model="username" placeholder="用户名">
    <input v-model="password" type="password" placeholder="密码">
    <button @click="handleLogin">登录</button>
    <p v-if="error">{{ error }}</p>
  </div>
</template>

<script setup>
import { ref } from 'vue';
import axios from 'axios';

const username = ref('');
const password = ref('');
const error = ref('');

async function handleLogin() {
    try {
        const res = await axios.post('/api/auth/login', {
            username: username.value,
            password: password.value
        });
        const token = res.data.data;
        localStorage.setItem('token', token);  // 存 token
        // 跳转到首页
        router.push('/home');
    } catch (e) {
        error.value = e.response?.data?.message || '登录失败';
    }
}
</script>
```

## 第二个坑：token 每次要带

每次请求都要带 `Authorization: Bearer xxx`，写在每个请求里很麻烦。

**正解：Axios 拦截器**

```javascript
// src/api/axios.js
import axios from 'axios';

const instance = axios.create({
    baseURL: '/api',
    timeout: 5000
});

// 请求拦截器：自动加 token
instance.interceptors.request.use(config => {
    const token = localStorage.getItem('token');
    if (token) {
        config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
});

// 响应拦截器：统一处理错误
instance.interceptors.response.use(
    response => response.data,  // 直接返回 data
    error => {
        if (error.response?.status === 401) {
            // token 失效，跳转登录
            localStorage.removeItem('token');
            router.push('/login');
        }
        return Promise.reject(error);
    }
);

export default instance;
```

然后所有 API 调用都用这个 `instance`：

```javascript
// src/api/user.js
import axios from './axios';

export function listUsers() {
    return axios.get('/users');  // 不用写 /api
}
```

**少了一堆重复代码**。

## 第三个坑：路由跳转

我登录成功后想跳到 `/home`，写了：

```javascript
import { useRouter } from 'vue-router';
const router = useRouter();
router.push('/home');
```

报错：

```
inject() can only be used inside setup() or functional components.
```

**原因**：`useRouter` 必须在 `setup()` 里调用，不能在普通函数里。

```javascript
// ❌ 错误
function handleLogin() {
    const router = useRouter();  // 这里错了
    router.push('/home');
}

// ✅ 正确
const router = useRouter();  // 在 setup 里
async function handleLogin() {
    // ...
    router.push('/home');
}
```

## 第四个坑：Vite 代理后端拿不到 cookie

登录用 cookie 存 session，前端带 `withCredentials`：

```javascript
axios.post('/api/auth/login', data, { withCredentials: true });
```

结果后端 session 还是空的。

**原因**：Vite 代理默认不带 cookie。

**解决**：

```javascript
// vite.config.js
server: {
    proxy: {
        '/api': {
            target: 'http://localhost:8080',
            changeOrigin: true,
            // 关键：保留 cookie
            cookieDomainRewrite: ''
        }
    }
}
```

或者干脆用 JWT，**不依赖 cookie**。

## 第五个坑：登录态保持

我登录后刷新页面，`localStorage` 的 token 还在，但 **Vue 状态丢了**——比如 `userInfo` 是 `null`。

**正解**：用 Pinia 存状态，初始化时从 localStorage 恢复。

```javascript
// src/stores/user.js
import { defineStore } from 'pinia';

export const useUserStore = defineStore('user', {
    state: () => ({
        token: localStorage.getItem('token') || '',
        userInfo: JSON.parse(localStorage.getItem('userInfo') || 'null')
    }),
    
    actions: {
        setUser(token, userInfo) {
            this.token = token;
            this.userInfo = userInfo;
            localStorage.setItem('token', token);
            localStorage.setItem('userInfo', JSON.stringify(userInfo));
        },
        
        logout() {
            this.token = '';
            this.userInfo = null;
            localStorage.removeItem('token');
            localStorage.removeItem('userInfo');
        }
    }
});
```

## 第六个坑：路由守卫

我想让没登录的用户访问不了 `/home`。

```javascript
// src/router/index.js
import { createRouter, createWebHistory } from 'vue-router';
import { useUserStore } from '../stores/user';

const router = createRouter({
    history: createWebHistory(),
    routes: [
        { path: '/login', component: () => import('../views/Login.vue') },
        { 
            path: '/home', 
            component: () => import('../views/Home.vue'),
            meta: { requiresAuth: true }
        }
    ]
});

router.beforeEach((to, from, next) => {
    const userStore = useUserStore();
    if (to.meta.requiresAuth && !userStore.token) {
        next('/login');
    } else {
        next();
    }
});

export default router;
```

## 完整的前后端联调流程

### 1. 后端接口规范

统一返回格式：

```java
@Data
public class Result<T> {
    private int code;       // 200 成功，500 失败
    private String message; // 消息
    private T data;         // 数据
}
```

后端接口：

```
POST /api/auth/login     登录
GET  /api/users          用户列表
GET  /api/users/{id}     用户详情
POST /api/users          创建用户
PUT  /api/users/{id}     更新用户
DELETE /api/users/{id}   删除用户
```

### 2. 前端目录结构

```
src/
├── api/                ← API 调用
│   ├── axios.js        ← axios 实例 + 拦截器
│   ├── user.js
│   └── auth.js
├── stores/             ← Pinia 状态
│   └── user.js
├── router/             ← 路由
│   └── index.js
├── views/              ← 页面
│   ├── Login.vue
│   ├── Home.vue
│   └── UserList.vue
├── components/         ← 组件
│   └── UserForm.vue
├── App.vue
└── main.js
```

### 3. 完整调用示例

登录后获取用户列表：

```vue
<!-- views/Home.vue -->
<template>
  <div>
    <h1>欢迎 {{ userStore.userInfo.username }}</h1>
    <button @click="handleLogout">退出</button>
    
    <UserList />
  </div>
</template>

<script setup>
import { useUserStore } from '../stores/user';
import { useRouter } from 'vue-router';
import UserList from '../components/UserList.vue';

const userStore = useUserStore();
const router = useRouter();

function handleLogout() {
    userStore.logout();
    router.push('/login');
}
</script>
```

```vue
<!-- components/UserList.vue -->
<template>
  <div>
    <button @click="loadUsers">刷新</button>
    <table>
      <tr>
        <th>ID</th>
        <th>用户名</th>
        <th>邮箱</th>
        <th>操作</th>
      </tr>
      <tr v-for="user in users" :key="user.id">
        <td>{{ user.id }}</td>
        <td>{{ user.username }}</td>
        <td>{{ user.email }}</td>
        <td>
          <button @click="deleteUser(user.id)">删除</button>
        </td>
      </tr>
    </table>
  </div>
</template>

<script setup>
import { ref, onMounted } from 'vue';
import { listUsers, deleteUser as delUser } from '../api/user';

const users = ref([]);

async function loadUsers() {
    try {
        users.value = await listUsers();
    } catch (e) {
        alert('加载失败：' + e.message);
    }
}

async function deleteUser(id) {
    if (!confirm('确定删除？')) return;
    try {
        await delUser(id);
        await loadUsers();  // 刷新列表
    } catch (e) {
        alert('删除失败：' + e.message);
    }
}

onMounted(loadUsers);
</script>
```

## 我做完后的几个最佳实践

### 1. 前后端分离的接口约定

- 后端：**永远返回统一格式** `{code, message, data}`
- 前端：**永远从 data 字段取数据**
- 错误码约定：200 成功，4xx 客户端错误，5xx 服务端错误

### 2. 跨域用 Vite 代理

开发环境用 `proxy`，**生产环境**才需要后端配 CORS（或者部署到同源）。

### 3. Axios 拦截器统一处理

- **请求拦截器**：加 token、加 loading 状态
- **响应拦截器**：401 跳登录、统一错误提示

### 4. Pinia 存状态

**别用 Vuex**（Vue 3 不用它了），**Pinia 是 Vue 3 官方推荐**。

### 5. 路由守卫做权限

- 路由 `meta.requiresAuth` 标记哪些页面要登录
- `beforeEach` 拦截，统一跳登录
