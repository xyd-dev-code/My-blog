## 起因

下学期要学 SpringBoot，老师留了个预习作业：

> "自己新建一个 SpringBoot 项目，浏览器访问 `localhost:8080/hello`，输出 hello world。"

我之前只写过 Java SE 和 Java Web（Servlet），**完全没碰过 Spring**。听说 SpringBoot 是"简化 Spring 的脚手架"，但简化成什么样没概念。

这篇把"从 0 到浏览器看到 hello"的全过程记下来。

## 第一步：在 IDEA 里新建 SpringBoot 项目

`File → New → Project → Spring Initializr`

填项目信息：

```
Name: hello-demo
Language: Java
Type: Maven
Group: com.example
Artifact: hello-demo
Package name: com.example.hellodemo
Java: 17
```

点 `Next`，选依赖页面：

我只选了一个：`Spring Web`（在 Web 分类下）。

> **Spring Web** 提供：
> - 内嵌 Tomcat
> - Spring MVC
> - RESTful 支持

不选这个，**就没有 Controller**。

点 `Finish`。IDEA 会下载依赖（第一次需要 5-10 分钟）。

## 第二步：项目结构

```
hello-demo/
├── pom.xml
├── src/
│   ├── main/
│   │   ├── java/
│   │   │   └── com/example/hellodemo/
│   │   │       └── HelloDemoApplication.java   ← 启动类
│   │   └── resources/
│   │       ├── application.properties          ← 配置文件
│   │       ├── static/                         ← 静态资源
│   │       └── templates/                      ← 模板
│   └── test/
│       └── java/
│           └── com/example/hellodemo/
│               └── HelloDemoApplicationTests.java
```

`HelloDemoApplication.java` 自动生成了：

```java
@SpringBootApplication
public class HelloDemoApplication {

    public static void main(String[] args) {
        SpringApplication.run(HelloDemoApplication.class, args);
    }
}
```

**就一个 main 方法**——这就是 SpringBoot 应用的入口。

## 第三步：写第一个 Controller

新建 `HelloController.java`：

```java
@RestController  // 返回字符串，不找视图
public class HelloController {

    @GetMapping("/hello")  // GET /hello
    public String hello() {
        return "Hello World!";
    }
}
```

两个注解：

- `@RestController`：告诉 Spring 这是个"返回数据的 Controller"
- `@GetMapping("/hello")`：路径是 `/hello`，方法是 GET

## 第四步：启动

两种方式：

**方式 1**：右键 `HelloDemoApplication.java` → Run

**方式 2**：IDEA 顶部的运行按钮

控制台会打印一堆日志，最后看到：

```
... Tomcat started on port(s): 8080 (http) ...
... Started HelloDemoApplication in 3.5 seconds ...
```

**看到 `Tomcat started` 就说明成功了**。

## 第五步：浏览器访问

打开浏览器，地址栏输入：

```
http://localhost:8080/hello
```

回车，页面显示：

```
Hello World!
```

我当时盯着屏幕看了三秒钟——**这也太简单了**。

要知道在学 SpringBoot 之前，Java Web 课要做的步骤是：

1. 新建 Maven Web 项目
2. 配置 `web.xml`
3. 写 Servlet 类继承 `HttpServlet`
4. 配置 `@WebServlet` 或 web.xml 路径
5. 打包成 war
6. 部署到外部 Tomcat
7. 启动 Tomcat
8. 浏览器访问

**8 步**，SpringBoot 把它简化成 **1 步**：跑 main 方法。

## 第六步：看一眼自动生成的 pom.xml

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project ...>
    <modelVersion>4.0.0</modelVersion>

    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>3.2.5</version>
        <relativePath/>
    </parent>

    <groupId>com.example</groupId>
    <artifactId>hello-demo</artifactId>
    <version>0.0.1-SNAPSHOT</version>

    <properties>
        <java.version>17</java.version>
    </properties>

    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>

        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-test</artifactId>
            <scope>test</scope>
        </dependency>
    </dependencies>

    <build>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>
</project>
```

**几个关键点**：

1. **`<parent>` 是 `spring-boot-starter-parent`**——这是 SpringBoot 的"父亲 POM"，里面定义了一堆默认配置（Java 版本、编码、依赖版本等）。
2. **`spring-boot-starter-web` 没有写 `<version>`**——版本由 parent 统一管理（3.2.5）。
3. **`spring-boot-maven-plugin`**——这个插件能把项目打包成"可执行 jar"，`java -jar xxx.jar` 就能跑。

## 几个我刚学时被搞混的概念

### 1. `@Controller` vs `@RestController`

```java
@Controller  // 返回视图名（找 templates/ 下的 html）
@RestController = @Controller + @ResponseBody  // 直接返回数据（JSON/字符串）
```

我现在的例子用 `@RestController`，**返回字符串直接给浏览器**。

如果用 `@Controller`，需要返回 html：

```java
@Controller
public class PageController {
    @GetMapping("/page")
    public String page() {
        return "hello";  // 找 templates/hello.html
    }
}
```

### 2. `@RequestMapping` vs `@GetMapping`

```java
@RequestMapping("/hello")           // 通用，支持所有方法
@GetMapping("/hello")               // 只支持 GET
@PostMapping("/hello")              // 只支持 POST
@PutMapping("/hello")               // 只支持 PUT
@DeleteMapping("/hello")            // 只支持 DELETE
```

`@GetMapping` 是 `@RequestMapping(method = RequestMethod.GET)` 的简写。

### 3. `application.properties` vs `application.yml`

`application.yml`（YAML 格式）：

```yaml
server:
  port: 8080
spring:
  application:
    name: hello-demo
```

`application.properties`（K-V 格式）：

```properties
server.port=8080
spring.application.name=hello-demo
```

两种格式都能用，看团队约定。**YAML 缩进要小心**（2 空格，不能用 Tab）。

## 改一下端口

`application.yml` 里加一行：

```yaml
server:
  port: 9090
```

重启项目，浏览器访问 `http://localhost:9090/hello`。

**搞定**——这就是改配置的威力。

## 返回 JSON 数据

```java
@RestController
public class HelloController {

    @GetMapping("/hello")
    public String hello() {
        return "Hello World!";
    }

    @GetMapping("/user")
    public Map<String, Object> user() {
        Map<String, Object> user = new HashMap<>();
        user.put("name", "Alice");
        user.put("age", 25);
        return user;  // 自动转 JSON
    }
}
```

访问 `http://localhost:8080/user`：

```json
{
  "name": "Alice",
  "age": 25
}
```

**返回对象自动转 JSON**——靠的是 Jackson 库，SpringBoot 默认引入。

### 用对象更优雅

```java
public class User {
    private String name;
    private int age;
    
    // getter/setter 省略
}

@GetMapping("/user")
public User user() {
    return new User("Alice", 25);
}
```

输出一样的 JSON。

> **用 Lombok** 简化 getter/setter：
> ```java
> @Data  // 自动生成 getter/setter/toString/equals
> public class User {
>     private String name;
>     private int age;
> }
> ```

## 接收参数

```java
@GetMapping("/hello")
public String hello(@RequestParam(name = "name", defaultValue = "World") String name) {
    return "Hello " + name + "!";
}
```

访问：

```
http://localhost:8080/hello?name=Alice
```

输出：

```
Hello Alice!
```

`@RequestParam` 的几个属性：

- `name` / `value`：参数名
- `defaultValue`：默认值
- `required`：是否必传（默认 true）

## 我当时没搞懂的两个问题

### 问题 1：为什么不写 web.xml？

**老 Spring 项目要写 web.xml**，注册 DispatcherServlet、CharacterEncodingFilter 等。

**SpringBoot 自动配置了这些**，所以不用写——这就是"约定优于配置"。

### 问题 2：Tomcat 哪来的？

我新建项目时**没装 Tomcat**。

**SpringBoot 内嵌了 Tomcat**（在 spring-boot-starter-web 依赖里），启动时自动跑起来。

部署时也不需要外部 Tomcat——项目打成 jar，`java -jar xxx.jar` 就跑起来了。

## 写一个最简单的测试

`src/test/java/.../HelloControllerTest.java`：

```java
@SpringBootTest
class HelloControllerTest {

    @Autowired
    private HelloController helloController;

    @Test
    void hello_返回HelloWorld() {
        String result = helloController.hello();
        assertEquals("Hello World!", result);
    }
}
```

但这种直接 new Controller 调用的测试意义不大，**真实项目里用 MockMvc**：

```java
@SpringBootTest
@AutoConfigureMockMvc
class HelloControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @Test
    void hello_返回HelloWorld() throws Exception {
        mockMvc.perform(get("/hello"))
               .andExpect(status().isOk())
               .andExpect(content().string("Hello World!"));
    }
}
```

`MockMvc` 模拟 HTTP 请求，不用真的启 Tomcat，**测试更快**。

## 一个让我当时懵的报错

启动时控制台打印：

```
***************************
APPLICATION FAILED TO START
***************************
Description:
Web server failed to start. Port 8080 was already in use.
```

**原因**：8080 端口被占用了（可能是之前启动的项目没关掉）。

**解决**：

**方式 1**：关掉之前占用 8080 的进程

```bash
# Windows 找占用 8080 的进程
netstat -ano | findstr :8080
# 看 PID 是什么，然后
taskkill /PID <PID> /F
```

**方式 2**：改端口（最简单）

```yaml
server:
  port: 8081
```

我选了方式 2，3 秒搞定。
